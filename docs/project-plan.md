# Plan du projet : TasteCam Heritage

## 1. Contexte
TasteCam Heritage est une plateforme web destinée à préserver et valoriser les recettes traditionnelles camerounaises, avec un focus sur les régions du Nord-Ouest et du Sud-Ouest.

## 2. Objectifs
- Créer un site de recettes culturelles facile à utiliser.
- Présenter l'origine culturelle de chaque plat.
- Permettre la recherche par ingrédient.
- Offrir une consultation en français et en anglais.
- Proposer une fonctionnalité de calcul des quantités selon le nombre de personnes.
- Montrer la région d'origine des plats.

## 3. Problème à résoudre
Les recettes traditionnelles du Cameroun sont souvent difficiles à trouver en ligne, avec peu d'informations sur leur histoire culturelle et leur région d'origine. TasteCam Heritage vise à regrouper ces informations dans une interface moderne, accessible et pédagogique.

## 4. Fonctionnalités principales
- Liste de recettes traditionnelles
- Détails d'une recette avec ingrédients, instructions et histoire
- Recherche par ingrédient
- Sélecteur de nombre de portions avec ajustement automatique des quantités
- Affichage de la région et de l'origine culturelle du plat
- Support des deux langues : français et anglais
- Médias associés : photos et liens vidéo

## 5. Architecture recommandée
- Frontend : React + Vite
- Backend : Node.js + Express
- Base de données : PostgreSQL
- Containerisation : Docker
- Orchestration : Kubernetes (étape suivante)
- CI/CD : Jenkins
- Monitoring : Prometheus + Grafana
- Infrastructure as Code : Ansible
