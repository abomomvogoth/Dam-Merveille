About the Project

A Platform for Culinary Storytelling

TasteCam-Heritage is a web-based application designed to digitize, preserve, and promote culinary heritage. Food is one of the most powerful expressions of culture and identity — yet traditional recipes risk being lost as communities modernize.

This project addresses that challenge by providing a structured, searchable, community-driven platform where users can discover, share, and archive traditional recipes from diverse cultural backgrounds, with a particular focus on African and Cameroonian culinary traditions.

System Design

Modern Layered Architecture

TasteCam-Heritage follows an N-Tier architecture with clear separation of concerns across presentation, business logic, and data layers.

Architectural Components

  • Frontend — Serves the user interface on port 4000, communicating via HTTP REST calls
  • Backend API — Node.js + Express.js on port 3000, handling all business logic and database operations
  • PostgreSQL Database — Stores recipes, categories, and user data with pg driver
  • Docker Containerization — Consistent environments for both frontend and backend services
  • Kubernetes Orchestration — 2-replica deployments with RollingUpdate and health probes
  • Jenkins CI/CD — Automated pipeline: Checkout → Install → Test → Build → Deploy
  • Prometheus & Grafana — Metrics collection and real-time dashboard visualization
  • Ansible IaC — Automated provisioning of VPS infrastructure
